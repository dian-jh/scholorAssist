# nacos配置

业务逻辑配置都要写到nacos中，因为敏感性、环境差异、动态性等等因素

docker中安装nacos

```java
docker run -d \
  --name nacos \
  -e MODE=standalone \
  -p 8848:8848 \
  -p 9848:9848 \
  -p 9849:9849 \
  -v /opt/nacos/logs:/home/nacos/logs \
  -v /opt/nacos/data:/home/nacos/data \
  nacos/nacos-server:v2.2.2
```

上面的是生产环境，在开发环境中，要使用外部数据库进行数据持久化

浏览器访问地址：

http://localhost:8848/nacos

yaml配置

```yaml
spring:
  application:
    name: sc-gateway
  profiles:
    active: dev
  config:
    import[0]: nacos:sc-gateway.yaml?group=DEFAULT_GROUP
#    spring.config.import[1]=nacos:{dataId1}?group={group1}
#	spring.config.import[2]=nacos:{dataId2}?group={group2}
  nacos:
    config:
    #        这个命名空间在这里要显示指定，我在这里浪费了好多时间 :(
      namespace: a5ddd2a9-c0a1-4885-b314-39a6fa2a480a
      server-addr: 127.0.0.1:8848
```
