# 文献辅助阅读系统 — PC端(Web)界面与核心功能需求

## 1. 页面结构与模块划分

- 登录/注册页面
  - 账号密码登录、注册、忘记/修改密码

- 主界面（经典三栏布局）
  - 左侧：类目树结构（多级文件夹，支持增删改拖拽与嵌套）
  - 中间：文献列表（包含上传、重命名、移动、删除、刷新等操作，支持搜索、排序）
  - 右侧：PDF阅读 & AI问答/笔记区
    - PDF预览与高亮选段
    - 选中文本弹出操作框（做笔记、AI问答、复制文本）
    - 历史AI回答与笔记列表同步显示

- 文献上传/解析弹窗
  - 可视化上传区
  - 解析中状态展示
  - 成功/失败反馈与自动刷新

- 用户设置页
  - 修改密码、退出登录、使用统计

## 2. 核心功能需求

- 用户账号体系
  - JWT鉴权，全局登录态管理
- 类目（文件夹）管理
  - 新增、重命名、删除、多级嵌套、拖拽排序（Tree结构）
- 文献管理
  - PDF上传、删除、重命名、分类调整、自动生成基本信息
  - 限制文件大小，显示解析状态
- 阅读器与AI&笔记区
  - 在线预览PDF，支持高亮、选择文字
  - 选定内容后弹出「笔记/AI/复制」快捷操作
  - 向AI提问（段落/全文），获取历史/当前回答
  - 增加/查看笔记，支持编辑、删除
- 数据同步
  - 账号下全数据（类目/文献/AI/笔记）云端存储，登录自动同步
- 交互体验
  - 上传进度、解析弹窗、操作延迟与反馈、AI回答卡片标记数据来源

## 3. 非功能性需求

- 安全性：全部接口需JWT权限校验
- 兼容性：桌面级浏览器适配（Chrome/Edge等）
- 稳定性：上传失败回滚，AI接口异常日志
- 性能：上传/AI任务异步处理，每用户不超5并发

## 4. 技术建议

- 框架：Vue3 + ElementPlus
- PDF预览：pdf.js
- 状态管理&通讯建议：Pinia/Store，WebSocket（笔记/AI实时同步）

## 5. 原型交互重点

- 三栏结构主界面
- 类目树编辑与拖拽流程
- PDF区域高亮/弹框交互完整演示
- 上传文件与解析进度流转
- AI/笔记侧栏同时操作与反馈
