# 文献辅助阅读系统 — 综合需求文档

## 一、项目定位

这是一个**文献辅助阅读系统**，旨在帮助用户高效管理、阅读并理解学术论文。
 用户可以在 Web 端或 H5 端：

- 分类存放文献资料（相当于文件夹层级管理）；
- 上传与浏览 PDF 文献；
- 在阅读时记录笔记；
- 选取文献片段进行 AI 问答或直接向 AI 提问；
- 登录账号后同步管理所有数据。

核心理念：

> 它是一个“智能文献文件夹”，集管理、阅读、思考为一体。

------

## 二、系统架构与平台

- **Web 端（主端）**：用于文献分类管理、上传、阅读、AI 问答。
- **H5 移动端**：轻量化版本，用于查看与同步笔记、AI 交互、阅读记录。
- **后端服务**：负责用户认证、文件管理、AI 接口、同步与存储。

------

## 三、系统主要功能（分端简要）

### 1. Web 端功能

主要面向学术用户的完整操作界面。

- **用户登录注册**
  - 支持账号密码登录、注册、修改密码。
  - 登录后进入用户专属文献空间。
- **文献类目管理**
  - 创建、重命名、删除类目（文件夹）。
  - 类目支持子文件夹（多级嵌套）。
  - 类目内可排序、拖拽调整。
- **文献管理**
  - 上传 PDF 文献（最大 50MB）。
  - 自动解析 PDF，生成标题、作者等基础信息。
  - 支持删除、重命名、移动到其他类目。
- **文献阅读器**
  - 在线预览 PDF。
  - 支持高亮选中内容。
  - 选中内容后可：
    1. 添加笔记（存储到数据库）。
    2. 直接向 AI 发问（如“解释这段的含义”）。
  - 在右侧显示 AI 回答历史与笔记区。
- **AI 辅助阅读**
  - 选中段落自动上下文问答。
  - 支持全篇提问（如“总结这篇论文的主要创新点”）。
  - 支持语义检索（跨文献搜索相关内容）。
- **同步系统**
  - 登录后，Web 与 H5 端同步笔记、AI 记录、文件结构。
- **用户设置**
  - 修改密码、退出登录、查看使用统计。

------

### 2. H5 移动端功能

为阅读和复习设计的轻量化版本，核心是同步与便捷交互。

- **用户登录**：与 Web 端共用账户体系。
- **同步内容**：
  - 获取所有类目结构。
  - 查看文献列表与AI问答历史。
  - 查看与编辑笔记。
- **AI 问答（移动版）**：
  - 可直接选择最近打开的文献与 AI 对话。
  - 或跨文献提问（例如“我有哪些论文提到 Transformer 模型？”）。
- **阅读器简版**：
  - 支持预览 PDF 页面或文本摘要。
  - 可标记重点、添加备注。

------

### 3. 后端功能（支撑所有端）

- 用户系统（JWT 认证、注册、登录、权限控制）
- 文件存储（PDF 上传、解析、存储路径管理）
- 文献结构管理（类目树）
- PDF 解析（文本抽取、切分、embedding 向量化）
- AI 模块（问答、摘要、推荐）
- 数据同步接口（Web ↔ H5）
- 日志与任务管理（上传状态、AI 调用记录）

------

## 四、核心使用流程（用户视角）

1. 用户注册并登录（Web 端）。
2. 创建“研究主题”文件夹，如「深度学习」「NLP」「CV」。
3. 上传一篇 PDF 到相应文件夹。
4. 系统后台解析文件 → 生成摘要与向量检索索引。
5. 用户打开该论文：
   - 阅读 PDF；
   - 选中段落 → 点击 “做笔记” 或 “AI 解释”。
6. AI 返回回答；用户可保存为笔记或收藏。
7. 用户在手机端（H5）登录：
   - 可浏览所有文件夹；
   - 查看自己的笔记和 AI 问答；
   - 继续提问或记录灵感。

## 五、系统模块划分（逻辑上）

| 模块           | 说明                     | 主要技术                              |
| -------------- | ------------------------ | ------------------------------------- |
| 用户模块       | 登录、注册、鉴权、同步   | Spring Security + JWT                 |
| 文献模块       | 文件上传、解析、摘要生成 | Spring Boot + PDFBox/Tika             |
| 类目模块       | 文件夹层级管理           | MyBatis / JPA                         |
| AI 模块        | 问答、摘要、推荐         | Spring AI + pgvector                  |
| 笔记模块       | 用户笔记记录与同步       | MyBatis + WebSocket/定期同步          |
| 移动端同步模块 | 同步用户资料与文献信息   | REST API                              |
| 文件存储模块   | 管理 PDF 文件路径与访问  | 本地 / OSS 存储                       |
| 前端模块       | Web + H5（UI层实现）     | Vue3 + Vant (H5) + Element Plus (Web) |

------

## 六、主要数据实体（简化版）

1. **用户（User）**
   - id, username, password_hash, created_at
2. **类目（Category）**
   - id, user_id, parent_id, name, created_at
3. **文献（Document）**
   - id, user_id, category_id, title, authors, storage_path, summary, uploaded_at
4. **文献分片（Chunk）**
   - id, document_id, text, embedding_vector
5. **笔记（Note）**
   - id, document_id, user_id, pdf_location, note_text, created_at
6. **问答记录（QA_Record）**
   - id, document_id, question, answer, created_at

------

## 七、前端原型开发重点（交互级）

### Web 端

- **主界面布局**
  - 左：类目树结构；
  - 中：文献列表；
  - 右：阅读/AI/笔记区。
- **阅读界面**
  - 支持高亮选择；
  - 弹出操作框（做笔记 / 问 AI / 复制文本）；
  - 底部显示 AI 回答。
- **上传流程**
  - 上传后显示“正在解析中”；
  - 后端异步任务完成后刷新状态。

### H5 端

- **首页：文件夹 → 文献列表**
- **文献详情页：摘要 + 最近笔记**
- **AI 对话界面：简洁上下文问答**
- **笔记列表：可编辑、同步状态提示**

## 八、后端接口（概览）

保持与前端清晰对接：

| 功能         | 接口                                  | 说明               |
| ------------ | ------------------------------------- | ------------------ |
| 登录         | `POST /api/auth/login`                | 获取 JWT           |
| 获取类目列表 | `GET /api/categories`                 | 支持树形结构       |
| 创建类目     | `POST /api/categories`                | 创建新类目         |
| 上传文献     | `POST /api/categories/{id}/documents` | PDF 上传           |
| 获取文献列表 | `GET /api/categories/{id}/documents`  | 按类目查询         |
| 获取文献详情 | `GET /api/documents/{id}`             | 包含摘要与状态     |
| 删除文献     | `DELETE /api/documents/{id}`          | 同时清除向量数据   |
| 选中文献问答 | `POST /api/documents/{id}/qa`         | AI 问答接口        |
| 保存笔记     | `POST /api/notes`                     | 保存笔记           |
| 获取笔记     | `GET /api/notes?documentId=xxx`       | 查询笔记           |
| 数据同步     | `GET /api/sync`                       | H5 与 Web 同步接口 |

## 九、非功能性需求

| 类型     | 要求                                     |
| -------- | ---------------------------------------- |
| 性能     | 单用户并发上传不超过 5，AI 调用异步化    |
| 安全     | 所有接口需 JWT 验证，文件上传大小 ≤ 50MB |
| 兼容性   | Web 端适配桌面浏览器，H5 端适配移动设备  |
| 稳定性   | 上传失败自动回滚，AI 异常需记录日志      |
| 用户体验 | 操作延迟提示明确，AI 回答需标识来源      |
