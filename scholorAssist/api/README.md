# 文献辅助阅读系统 API 文档

## 概述

本文档详细描述了文献辅助阅读系统的所有API接口，包括请求参数、响应格式、错误处理等信息。

## 技术规范

- **基础URL**: `/api`
- **请求方式**: 仅支持 GET 和 POST
- **数据格式**: JSON
- **字符编码**: UTF-8
- **认证方式**: Bearer Token（部分接口需要）

## 统一响应格式

所有API接口均采用统一的响应格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

### 响应字段说明

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| code | int | 是 | 状态码（200表示成功，非200表示错误） |
| msg | string | 是 | 状态信息（成功时为"success"，错误时为具体错误描述） |
| data | any | 是 | 实际返回的业务数据（可为null） |

## 错误码说明

| 状态码 | 说明 | 使用场景 |
|-------|------|---------|
| 200 | 成功 | 请求处理成功 |
| 400 | 请求参数错误 | 参数格式不正确、必填参数缺失等 |
| 401 | 未授权 | 需要登录或token无效 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | 请求的资源未找到 |
| 409 | 资源冲突 | 数据已存在或状态冲突 |
| 500 | 服务器内部错误 | 系统异常 |
