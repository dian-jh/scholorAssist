# 文献辅助阅读系统 — 综合需求文档

## 一、项目定位

这是一个**文献辅助阅读系统**，旨在帮助用户高效管理、阅读并理解学术论文。
 用户可以在 Web 端或 H5 端：	

- 分类存放文献资料（相当于文件夹层级管理）；
- 上传与浏览 PDF 文献；
- 在阅读时记录笔记；
- 选取文献片段进行 AI 问答或直接向 AI 提问；
- 登录账号后同步管理所有数据。

核心理念：

> 它是一个“智能文献文件夹”，集管理、阅读、思考为一体。

------

## 二、系统架构与平台

- **Web 端（主端）**：用于文献分类管理、上传、阅读、AI 问答。
- **H5 移动端**：轻量化版本，用于查看与同步笔记、AI 交互、阅读记录。
- **后端服务**：负责用户认证、文件管理、AI 接口、同步与存储。

------

## 三、系统主要功能（分端简要）

### 1. Web 端功能

主要面向学术用户的完整操作界面。

- **用户登录注册**
  - 支持账号密码登录、注册、修改密码。
  - 登录后进入用户专属文献空间。
- **文献类目管理**
  - 创建、重命名、删除类目（文件夹）。
  - 类目支持子文件夹（多级嵌套）。
  - 类目内可排序、拖拽调整。
- **文献管理**
  - 上传 PDF 文献（最大 50MB）。
  - 自动解析 PDF，生成标题、作者等基础信息。
  - 支持删除、重命名、移动到其他类目。
- **文献阅读器**
  - 在线预览 PDF。
  - 支持高亮选中内容。
  - 选中内容后可：
    1. 添加笔记（存储到数据库）。
    2. 直接向 AI 发问（如“解释这段的含义”）。
  - 在右侧显示 AI 回答历史与笔记区。
- **AI 辅助阅读**
  - 选中段落自动上下文问答。
  - 支持全篇提问（如“总结这篇论文的主要创新点”）。
  - 支持语义检索（跨文献搜索相关内容）。
- **同步系统**
  - 登录后，Web 与 H5 端同步笔记、AI 记录、文件结构。
- **用户设置**
  - 修改密码、退出登录、查看使用统计。



------

### 3. 后端功能（支撑所有端）

- 用户系统（JWT 认证、注册、登录、权限控制）
- 文件存储（PDF 上传、解析、存储路径管理）
- 文献结构管理（类目树）
- PDF 解析（文本抽取、切分、embedding 向量化）
- AI 模块（问答、摘要、推荐）
- 数据同步接口（Web ↔ H5）
- 日志与任务管理（上传状态、AI 调用记录）

------

## 四、核心使用流程（用户视角）

1. 用户注册并登录（Web 端）。
2. 创建“研究主题”文件夹，如「深度学习」「NLP」「CV」。
3. 上传一篇 PDF 到相应文件夹。
4. 系统后台解析文件 → 生成摘要与向量检索索引。
5. 用户打开该论文：
   - 阅读 PDF；
   - 选中段落 → 点击 “做笔记” 或 “AI 解释”。
6. AI 返回回答；用户可保存为笔记或收藏。
7. 用户在手机端（H5）登录：
   - 可浏览所有文件夹；
   - 查看自己的笔记和 AI 问答；
   - 继续提问或记录灵感。

## 五、系统模块划分（逻辑上）

| 模块           | 说明                     | 主要技术                              |
| -------------- | ------------------------ | ------------------------------------- |
| 用户模块       | 登录、注册、鉴权、同步   | Spring Security + JWT                 |

| 移动端同步模块 | 同步用户资料与文献信息   | REST API                              |
| 文件存储模块   | 管理 PDF 文件路径与访问  | 本地 / OSS 存储                       |
| 前端模块       | Web + H5（UI层实现）     | Vue3 + Vant (H5) + Element Plus (Web) |

## 六、主要数据实体（简化版）

1. **用户（User）**
   - id, username, password_hash, created_at
2. **类目（Category）**
   - id, user_id, parent_id, name, created_at
3. **文献（Document）**
   - id, user_id, category_id, title, authors, storage_path, summary, uploaded_at
4. **文献分片（Chunk）**
   - id, document_id, text, embedding_vector
5. **笔记（Note）**
   - id, document_id, user_id, pdf_location, note_text, created_at
6. **问答记录（QA_Record）**
   - id, document_id, question, answer, created_at

# ai模块

文献总结设计：层次化摘要

原始文献 (可能几千字~几十万字)

 ⬇ 分块 (chunking)

得到 N 个 chunk

 ⬇ 对每个 chunk 做“局部总结”

得到 N 个 小总结

 ⬇ 将小总结再组合成 3~10段中间总结

得到 M 个中层总结

 ⬇ 最后对中层总结再总结一次

得到 全文 Summary

将summary写入向量数据库，

对用户输入的prompt进行意图判断，使用一个比较小的/廉价的LLM来分类意图，总结/详情，输入prompt，只返回 "summary" 或 "detail"。如果返回 `summary` → **直接查 metadata type=summary**
 如果返回 `detail` → **进行向量检索**

### 举几个例子：

| 用户提问                                 | 意图    | 系统行为                             |
| ---------------------------------------- | ------- | ------------------------------------ |
| “这篇论文讲了什么？”                     | summary | 直接返回存储的文献 summary           |
| “这篇文献的创新点在哪里？”               | summary | 返回 summary，因为创新点通常在总论里 |
| “第三章为什么要使用 Transformer？”       | detail  | 进行向量检索                         |
| “论文中有没有提到损失函数的稳定性问题？” | detail  | 相似度搜索相关 chunk                 |
